# Paper plan

The arxiv preprint that accompanies the ACP reference implementation lives
in `paper/`.

## Scope

Single-author preprint, ~6 pages, narrative-driven. The point is not
academic novelty (ACP is engineering) but to establish citeable prior art
and a fixed timestamp for the protocol design before the public launch.

## Source of truth

- `paper/acp.md` is the canonical text. Easy to read in a PR, easy to
  update with new measurements.
- `paper/acp.tex` is hand-mirrored from `acp.md` and is what gets
  submitted to arxiv.
- The two are kept in sync by `scripts/check_paper.py` (see CI gates).

## Charts

Generated by `python3 benchmark/charts.py --in
results/<dated>-baseline.json --out-dir results/charts` and then copied
into `paper/figures/`. The script uses matplotlib's Agg backend so it
works headless in CI.

## CI workflow

`.github/workflows/paper.yml` runs on every PR that touches `paper/`,
`benchmark/charts.py`, or `results/`:

1. Generate charts from the latest committed results JSON.
2. Verify charts in `paper/figures/` match `results/charts/`.
3. Build `paper/acp.pdf` inside the official TeX Live container.
4. Upload `acp.pdf` as a build artifact for review.

## Submission timeline

| Step | When |
|---|---|
| Final review of `acp.md` and `acp.tex` | After the public spec launch (Phase 5) |
| Build PDF, double-check figures | Same day |
| Submit to arxiv (cs.SE primary, cs.LG cross-list) | Within 24h of public launch |
| Tweet announcement linking arxiv ID | Same day |
| Reference arxiv ID in YC/a16z applications | Ongoing |

## arxiv submission checklist

- `acp.tex`, `references.bib`, `figures/*.png` (PDF embeds need PNG).
- Abstract under 1920 characters.
- Categories: cs.SE primary; cs.LG, cs.AI cross-list.
- License: arxiv.org perpetual + non-exclusive distribution; ACP spec
  remains CC BY 4.0.
- Author endorsement: not needed for the chosen categories.

produce updated S4 numbers before submitting? Decision: **no** -- the
