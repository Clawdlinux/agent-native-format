## 1. Abstract

**ACL (Agent Context Language)** is a line-oriented, machine-native textual
representation designed to be **consumed by LLM agents**, not by humans.

Existing systems — REST APIs, Kubernetes resources, MCP `tools/list`
payloads, Slack messages, web pages — encode their state in formats
optimised for human perception (deep JSON, HTML, prose). When an LLM agent
ingests these, the majority of its context window is spent parsing
*format*, not *substance*: nesting brackets, repeated field names, escaped
strings, decorative whitespace, JSON Schema descriptions of entities the
model already understands.

ACL inverts that. It is the **target representation** that an agent sees
after a *Translator* has converted a human-format source. ACL is:

- **Compact** — typically 10–40× fewer tokens than equivalent JSON
- **Tokenizer-friendly** — pure ASCII, short reserved tokens
- **Stable** — deterministic encoding; identical input → identical bytes
- **Round-trippable** — `Decode(Encode(d)) == d`
- **Domain-agnostic** — Kubernetes, SaaS APIs, web pages all encode the
  same way; only the section nouns and field keys change
- **Lossless within scope** — the translator decides what to include;
  whatever it includes is preserved byte-for-byte

ACL is **not** a wire protocol. It is a representation. The wire protocol
that delivers ACL documents is [ACP](../SPEC.md). The pipeline is:

```
[ source ] → [ Translator ] → [ ACL document ] → [ ACP transport ] → [ agent ]
  (human                        (machine-native)   (HTTP/SSE)
   format)
```

## 2. Design goals

1. **Optimise for the consumer.** The consumer is an LLM. Bytes that don't
   change the agent's decision are wasted bytes.
2. **Eliminate format tax.** No quotes around bare identifiers. No commas
   between fields. No type annotations the agent already infers.
3. **Preserve affordances.** A representation must declare what actions
   are possible, not just what state exists.
4. **Encoding is mechanical, decoding is mechanical.** A translator and an
   agent must agree on the bytes without negotiation.
5. **Stable hashing.** Two identical states produce identical documents,
   so caches and content-addressable stores work.

Non-goals: human readability (a side effect, not a goal); binary efficiency
(LLM tokenisers tokenise binary poorly — staying ASCII is the right
trade-off for the consumer).

## 3. Lexical structure

A document is a UTF-8 byte sequence (ASCII subset for v0.1) divided into
**lines** terminated by `\n` (LF). Trailing whitespace on a line is
significant only inside quoted values (see §6).

### 3.1 Line kinds

| Prefix             | Kind                  | Example                                 |
|--------------------|-----------------------|-----------------------------------------|
| `# `               | Comment (stripped)    | `# generated by k8s-translator v0.1`    |
| `@`                | Directive             | `@cluster prod-east`                    |
| (empty)            | Section break         |                                         |
| no leading space   | Section header        | `pods 12/12 ok`                         |
| 2 leading spaces   | Row                   | `  payment-api-7f8d x3 cpu=42 r=0`      |

Indentation is exactly **two spaces**. Tabs are rejected. Nested rows are
not supported in v0.1 (use cross-section refs instead, §6.4).

### 3.2 Reserved tokens

| Token         | Meaning                                                  |
|---------------|----------------------------------------------------------|
| `!`           | Warning marker on a row                                  |
| `!!`          | Critical marker on a row                                 |
| `?`           | Unknown / unreadable value                               |
| `x<int>`      | Instance count for the row's identifier (e.g. `x3`)      |
| `=`           | Field key/value separator                                |
| `\|`          | Affordance separator in `actions` rows                   |
| `->`          | Mapping (port `8080->8080`, ref `deploy->pod`)           |

These tokens **must not** appear unquoted inside identifiers or values.
Quote with backticks if needed: `` cmd=`echo hi | wc -l` ``.

## 4. Document structure

```
[header-comment]?
[directives]*
[section]+
```

A document is a sequence of zero-or-more directives followed by one-or-more
sections, separated by blank lines.

### 4.1 Directives

A directive is `@<key> <value>` on a single line. Directives carry
document-scope metadata: the source, the time, the scope.

```
@cluster prod-east
@ns payments
@as-of 2026-05-03T12:34:56Z
@source k8s/v1
```

Reserved directive keys (more may be added):
- `@cluster`, `@ns`, `@source`, `@as-of`, `@id`, `@hash`, `@translator`

### 4.2 Sections

A section is a header line followed by zero-or-more rows, terminated by a
blank line or end-of-file:

```
<noun> [summary]
  <row>
  <row>
```

The **noun** is a short, lowercase identifier (`pods`, `deploys`, `svcs`,
`alerts`, `actions`). The optional **summary** is a short aggregate
(`12/12 ok`, `3 all-avail`, `2`) — included so the agent can short-circuit
when the summary is sufficient.

### 4.3 Rows

A row is `  <id> [x<count>] [field|flag]*`:

```
  payment-api-7f8d x3 cpu=42 mem=61 r=0 age=3d
  payment-worker-9a2b x2 cpu=87 mem=78 r=3 age=1d !
```

- `<id>` is an unquoted identifier matching `[A-Za-z0-9_./:-]+`
- `x<count>` (optional) is the instance count (replicas, occurrences)
- Each subsequent space-separated token is either a **field**
  (`key=value`) or a **flag** (`!`, `!!`, `?`, or any short bareword)

## 5. Special section: `actions`

The `actions` section declares the affordances available to the agent in
the current scope:

```
actions
  scale|rollout|restart|logs|describe
```

A single row, pipe-separated. Agents read this as the closed set of verbs
they may invoke. Capabilities outside this set are not available.

## 6. Values

### 6.1 Bare values

Bare values match `[A-Za-z0-9_./:%@+-]+` and need no quoting.

### 6.2 Quoted values

Values containing reserved tokens, spaces, or non-ASCII bytes must be
backtick-quoted: `` cmd=`tail -n 100 /var/log/nginx.log` ``. Backticks
inside values are escaped as `` `` ``.

### 6.3 Numeric and unit values

Numbers are bare: `cpu=42`, `mem=61`. Units are appended without a space
when conventional in the domain: `age=3d`, `at=78%`. Translators **should**
strip percent signs where the field name implies it (`cpu=42` not
`cpu=42%`).

### 6.4 References

Cross-section references use the noun-singular form as a key:
`pod=payment-worker-9a2b`. The agent resolves these by looking up the
identifier in the corresponding section. This is how ACL avoids deep
nesting.

## 7. Worked example — Kubernetes namespace

```
# generated by acl-k8s/v0.1
@cluster prod-east
@ns payments
@as-of 2026-05-03T12:34:56Z

pods 12/12 ok
  payment-api-7f8d x3 cpu=42 mem=61 r=0 age=3d
  payment-worker-9a2b x2 cpu=87 mem=78 r=3 age=1d !
  payment-cron-1c4e x1 cpu=5 mem=12 r=0 age=7d

deploys 3 all-avail
  payment-api 3/3 rolling v2.4.1

svcs 2
  payment-api ClusterIP 8080->8080

alerts 1
  HighMemory pod=payment-worker-9a2b at=78 lim=75

actions
  scale|rollout|restart|logs|describe
```

Token cost (cl100k_base): **~140 tokens**. The equivalent
`kubectl get pods,deploy,svc -n payments -o json` for the same state is
**~8,200 tokens**. Compression: **~58×**.

## 8. Round-trip and stability guarantees

A conformant encoder MUST:
- Emit directives in insertion order.
- Emit sections in insertion order.
- Emit rows within a section in insertion order.
- Emit fields within a row in insertion order.
- Use a single space as field separator and `\n` as line terminator.
- Never emit trailing whitespace on a line.

A conformant decoder MUST:
- Strip `# `-prefixed comment lines (preserve them as `Document.Comments`
  if needed for debug; not part of equality).
- Reject tabs as indentation.
- Treat blank lines only as section terminators (no semantic content).

Given these rules, `Decode(Encode(d)) == d` for any well-formed `d`.

## 9. Versioning

This document is **ACL v0.1**. Future versions may:
- Add reserved directive keys (backward compatible)
- Add reserved tokens (breaking — bumps minor)
- Change indentation rules (breaking — bumps major)

Documents may declare their version with `@acl 0.1`. Decoders SHOULD treat
absence as `0.1` for now.

## 10. Reference implementation

The Go reference encoder/decoder lives at
[`pkg/acl`](../pkg/acl/). Conformance is asserted by:
- Round-trip tests (`acl_test.go`)
- Golden-file fixtures (`testdata/`)
- Token-count benchmarks against equivalent JSON (`bench_test.go`)

Bindings in other languages are out of scope for v0.1; the format is
simple enough (~150 LOC) that re-implementation is preferred over a shared
runtime.

## 11. Relationship to ACP

ACL is the **payload**. ACP (see [SPEC.md](../SPEC.md)) is the **wire
protocol** that requests, delivers, and feeds back on ACL documents.
The existing ACP "Execution Manifest" is one specific *typed view*
rendered in ACL — see [`docs/positioning.md`](./positioning.md) §3 for the
full layered model.
