ACP — Agent Context Protocol

One API call. Complete execution context. Minimal tokens. Sits on top of MCP.

MCP answers "what tools exist?". ACP sits on top of MCP and answers "what exactly do I need to do right now, and how?". The agent submits an intent; ACP returns one intent-scoped Execution Manifest with auth pre-injected, dependency order pre-computed, and security boundaries declared. Existing MCP servers keep working.

Try it (~30 seconds)

git clone https://github.com/Clawdlinux/ninevigil-acp
cd ninevigil-acp
go build -o bin/acp-server ./cmd/acp-server
ACP_AUTH_TOKEN=dev ./bin/acp-server --addr :8080 &

curl -sS -X POST http://localhost:8080/v1/context \
  -H "Authorization: Bearer dev" \
  -H "Content-Type: application/json" \
  -d '{"intent":"query the customer database, render a report, email the team",
       "agent_id":"demo"}' | python3 -m json.tool

Use it from your stack

Python (OpenAI)

from openai import OpenAI
from acp_common import ACPClient
from acp_openai import manifest_to_openai_tools, dispatch_tool_call

acp = ACPClient("http://localhost:8080", auth_token="dev")
manifest = acp.context(intent="send last week's report", agent_id="analyst")
tools = manifest_to_openai_tools(manifest)

resp = OpenAI().chat.completions.create(
    model="gpt-4o-mini", tools=tools,
    messages=[{"role": "user", "content": "send the report"}],
)

Go

client := acp.NewClient("http://localhost:8080", acp.WithToken("dev"))
mf, _ := client.Context(ctx, manifest.ContextRequest{
    Intent:  "query the customer db",
    AgentID: "agent-01",
})
// mf.Actions is in topological order; mf.Boundaries enforces egress.

Any language

It's HTTP/JSON. POST /v1/context, parse the manifest, execute via the proxy.

What you get

  • Intent-scoped manifests. Only the tools relevant to the task. Schemas compacted into a typed mini-language.
  • Auth never enters the agent context window. Credentials are injected at a proxy boundary.
  • Deterministic execution order. depends_on is server-computed. The agent doesn't reason about ordering.
  • Declared security boundaries. Egress allow-list, per-action token ceiling, approval gates, audit level — all part of the contract.
  • MCP-compatible. Point ACP at any MCP server. The reference implementation ingests MCP tools/list payloads automatically.
  • Open spec. CC BY 4.0. SDKs Apache 2.0. Reference runtime BSL 1.1 with 3-year Apache conversion.
